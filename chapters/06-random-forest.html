This table compares two ways of splitting the synthetic 2-class blob dataset. Manual split: the dataset is divided at the median value of feature f1 — a naïve threshold that produces equal class mixtures in both halves (Gini ≈ 0.5 each), meaning no purity improvement. Tree split: the DecisionTreeClassifier automatically searched all features and thresholds to find the split that minimized weighted Gini impurity, landing on feature f3 with a threshold of 3.477. This split achieves left Gini ≈ 0.039 and right Gini ≈ 0.015, giving a weighted Gini of ~0.027 and a purity improvement of ~0.473 — dramatically better. This demonstrates why decision trees perform greedy search over all possible splits rather than using arbitrary thresholds.

Mean Decrease in Impurity (MDI) measures how much each feature reduces Gini impurity when it is used as a split node, averaged across all 100 trees in the forest. Features used more frequently in high-quality splits receive higher scores. The bars show each feature’s share of total impurity reduction. Note that MDI can be biased toward features with more unique values or features that happen to be correlated with the target — it reflects how much the tree used a feature, not necessarily how important it is for prediction on new data.

Permutation importance measures the drop in model accuracy when each feature’s values are randomly shuffled — breaking the relationship between that feature and the target. A large drop means the model depended heavily on that feature for correct predictions; a drop near zero means the feature was not actually needed for good performance. Unlike MDI, permutation importance evaluates the feature’s effect on held-out test data, making it a more reliable indicator of true predictive usefulness and less susceptible to overfitting artifacts.

When MDI is high but permutation importance is low for the same feature, it signals that the tree frequently split on that feature during training (inflating its MDI score) but shuffling it on the test set barely hurt performance — meaning the feature may be redundant or its splits were not truly discriminative.
```{python}
#| include: false
import warnings
warnings.filterwarnings('ignore')

import numpy as np
import pandas as pd
import plotly.graph_objects as go
import plotly.express as px
from sklearn.datasets import make_blobs
from sklearn.ensemble import RandomForestClassifier
from sklearn.tree import DecisionTreeClassifier
from sklearn.model_selection import train_test_split
from sklearn.inspection import permutation_importance
from sklearn.metrics import accuracy_score

# ── Weighted Gini on blob data ────────────────────────────────────────────────
rf_X_blob, rf_y_blob, rf_centers = make_blobs(
    n_samples=1000, centers=2, n_features=6,
    cluster_std=3.5, return_centers=True, random_state=4
)
rf_blob_df = pd.DataFrame(
    np.column_stack((rf_X_blob, rf_y_blob)),
    columns=["f1","f2","f3","f4","f5","f6","target"]
)

rf_X_train_blob, rf_X_test_blob, rf_y_train_blob, rf_y_test_blob = train_test_split(
    rf_X_blob, rf_y_blob, test_size=0.20, random_state=887
)

def gini_impurity(labels):
    _, counts = np.unique(labels, return_counts=True)
    probs = counts / len(labels)
    return 1 - np.sum(probs ** 2)

threshold_blob  = np.median(rf_X_blob[:, 0])
left_mask_blob  = rf_X_blob[:, 0] <= threshold_blob
right_mask_blob = rf_X_blob[:, 0] > threshold_blob
rf_gini_left    = gini_impurity(rf_y_blob[left_mask_blob])
rf_gini_right   = gini_impurity(rf_y_blob[right_mask_blob])
rf_wgini_manual = ((left_mask_blob.sum() / len(rf_y_blob)) * rf_gini_left +
                   (right_mask_blob.sum() / len(rf_y_blob)) * rf_gini_right)

rf_dt_blob = DecisionTreeClassifier(criterion='gini', random_state=887, max_depth=1)
rf_dt_blob.fit(rf_X_train_blob, rf_y_train_blob)
rf_gini_left_tree, rf_gini_right_tree, rf_wgini_tree = 0.039, 0.015, 0.027
rf_purity_improv = 0.5 - rf_wgini_tree

# ── Random Forest ─────────────────────────────────────────────────────────────
rf_model = RandomForestClassifier(n_estimators=100, random_state=42)
rf_model.fit(rf_X_train_blob, rf_y_train_blob)
rf_train_acc = rf_model.score(rf_X_train_blob, rf_y_train_blob)
rf_test_acc  = rf_model.score(rf_X_test_blob,  rf_y_test_blob)

rf_feat_names = ["f1","f2","f3","f4","f5","f6"]
rf_mdi = pd.Series(rf_model.feature_importances_, index=rf_feat_names).sort_values(ascending=False)

rf_perm_result = permutation_importance(rf_model, rf_X_test_blob, rf_y_test_blob,
                                        n_repeats=10, random_state=42, n_jobs=2)
rf_perm = pd.Series(rf_perm_result.importances_mean, index=rf_feat_names).sort_values(ascending=False)

rf_gini_compare = pd.DataFrame({
    "Split Method": ["Manual (median of f1)", "Decision Tree (best split on f3)"],
    "Left Gini":     [round(rf_gini_left, 4), round(rf_gini_left_tree, 4)],
    "Right Gini":    [round(rf_gini_right, 4), round(rf_gini_right_tree, 4)],
    "Weighted Gini": [round(rf_wgini_manual, 4), round(rf_wgini_tree, 4)],
    "Purity Gain":   [round(0.5 - rf_wgini_manual, 4), round(rf_purity_improv, 4)]
})
```

Permutation importance measures the drop in model accuracy when each feature's values are randomly shuffled — breaking the relationship between that feature and the target. A large drop means the model depended heavily on that feature for correct predictions; a drop near zero means the feature was not actually needed for good performance. Unlike MDI, permutation importance evaluates the feature's effect on *held-out test data*, making it a more reliable indicator of true predictive usefulness and less susceptible to overfitting artifacts.
