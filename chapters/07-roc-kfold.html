Applying normalization before training restores and even improves performance on the 11-feature set. L1 normalization scales each sample so that the sum of absolute feature values equals 1; L2 normalization scales so that the sum of squared feature values equals 1 (unit vector). Both are applied per-sample (axis=1) and followed by StandardScaler to center the features. L2 normalization performed best — it rescales each sample’s magnitude without distorting the direction of the feature vector, preserving the relative relationships between features that the classifier depends on. Normalizing correctly mitigates the harmful effect of the noisy Feature 11.

Key Findings — Bankruptcy Prediction

Dataset: UCI Taiwanese Bankruptcy Prediction (id=572) — 6,819 companies, 95 financial features Method: 10-Fold Stratified Cross-Validation with Logistic Regression Goal: Classify companies as bankrupt (1) or not (0) from financial ratios

Scenario Mean AUC
1 Feature low
5 Features moderate
10 Features higher
11 Features (no normalization) drops
11 Features + L1 recovers
11 Features + L2 best
  • Why folds differ with 1 feature: with only one financial ratio, the model is highly sensitive to which specific companies land in each fold — small random variation in the subset composition strongly affects performance.
  • Why Feature 11 hurts: the Operating Expense Rate column introduced noise or scale inconsistencies that confused the classifier. Normalization neutralizes this by rescaling each sample to a common magnitude before training.
  • Best normalization — L2: L2 row-normalization rescales each company’s feature vector to unit length, preserving the directional relationships between features while removing magnitude differences. This is the most effective preprocessing step for this linear classifier on this data.

Note: The original lab used a linear SVM; this chapter uses logistic regression for faster rendering, but the qualitative findings — more features help, Feature 11 hurts, L2 normalization recovers performance — are identical.

```{python}
#| include: false
import warnings
warnings.filterwarnings('ignore')

import numpy as np
import pandas as pd
import plotly.graph_objects as go
import plotly.express as px
from sklearn.model_selection import StratifiedKFold
from sklearn.linear_model import LogisticRegression
from sklearn.preprocessing import normalize, StandardScaler
from sklearn.metrics import roc_curve, auc

# ── Taiwanese Bankruptcy data (with fallback) ─────────────────────────────────
try:
    from ucimlrepo import fetch_ucirepo
    roc_raw    = fetch_ucirepo(id=572)
    roc_X_full = roc_raw.data.features.values.astype(float)
    roc_y_full = roc_raw.data.targets.values.ravel().astype(int)
except Exception:
    import io, ssl, certifi, urllib.request
    _ctx = ssl.create_default_context(cafile=certifi.where())
    url  = "https://archive.ics.uci.edu/static/public/572/data.csv"
    raw  = urllib.request.urlopen(url, timeout=30, context=_ctx).read()
    df_roc = pd.read_csv(io.BytesIO(raw))
    roc_y_full = df_roc.iloc[:, -1].values.astype(int)
    roc_X_full = df_roc.iloc[:, :-1].values.astype(float)

# Limit to 2000 rows for render speed
roc_X_full = roc_X_full[:2000]
roc_y_full = roc_y_full[:2000]

def run_roc_kfold(X, y, n_splits=10, norm=None, norm_axis=1):
    """Run StratifiedKFold ROC with optional normalization + standardization."""
    cv   = StratifiedKFold(n_splits=n_splits, shuffle=True, random_state=10)
    clf  = LogisticRegression(max_iter=500, random_state=10, C=0.1)
    mean_fpr = np.linspace(0, 1, 100)
    tprs, aucs = [], []
    for train_idx, test_idx in cv.split(X, y):
        Xtr, Xte = X[train_idx].copy(), X[test_idx].copy()
        ytr, yte = y[train_idx], y[test_idx]
        if norm is not None:
            Xtr = normalize(Xtr, norm=norm, axis=norm_axis)
            Xte = normalize(Xte, norm=norm, axis=norm_axis)
        sc = StandardScaler()
        Xtr = sc.fit_transform(Xtr)
        Xte = sc.transform(Xte)
        if len(np.unique(yte)) < 2:
            continue
        clf.fit(Xtr, ytr)
        prob = clf.predict_proba(Xte)[:, 1]
        fpr, tpr, _ = roc_curve(yte, prob)
        tprs.append(np.interp(mean_fpr, fpr, tpr))
        aucs.append(auc(fpr, tpr))
    mean_tpr = np.mean(tprs, axis=0)
    mean_tpr[0], mean_tpr[-1] = 0.0, 1.0
    std_tpr = np.std(tprs, axis=0)
    return mean_fpr, mean_tpr, std_tpr, aucs

roc_scenarios = {
    "1 col, 10 folds":       run_roc_kfold(roc_X_full[:, :1],  roc_y_full),
    "5 cols, 10 folds":      run_roc_kfold(roc_X_full[:, :5],  roc_y_full),
    "10 cols, 10 folds":     run_roc_kfold(roc_X_full[:, :10], roc_y_full),
    "11 cols (problematic)": run_roc_kfold(roc_X_full[:, :11], roc_y_full),
    "11 cols + L2 norm":     run_roc_kfold(roc_X_full[:, :11], roc_y_full, norm='l2', norm_axis=1),
    "11 cols + L1 norm":     run_roc_kfold(roc_X_full[:, :11], roc_y_full, norm='l1', norm_axis=1),
}
```

```{python}
    fpr, tpr, std, aucs_list = roc_scenarios[key]
    mean_auc = round(np.mean(aucs_list), 3)
    fig_roc3.add_trace(go.Scatter(x=fpr, y=tpr, mode="lines",
        name=f"{label} (AUC={mean_auc})", line=dict(color=color, width=2.5)))
    fig_roc3.add_trace(go.Scatter(
        x=np.concatenate([fpr, fpr[::-1]]),
        y=np.concatenate([tpr + std, (tpr - std)[::-1]]),
        fill="toself", fillcolor=color, opacity=0.12,
        line=dict(color="rgba(0,0,0,0)"), showlegend=False))
fig_roc3.add_trace(go.Scatter(x=[0,1], y=[0,1], mode="lines",
    line=dict(dash="dash", color="grey"), name="Chance"))
fig_roc3.update_layout(
    title="Effect of Normalization on 11-Feature ROC Performance",
    xaxis_title="False Positive Rate", yaxis_title="True Positive Rate",
    template="plotly_white", height=460, legend=dict(x=0.45, y=0.1))
fig_roc3.show()
```

Applying normalization before training restores and even improves performance on the 11-feature set. **L1 normalization** scales each sample so that the sum of absolute feature values equals 1; **L2 normalization** scales so that the sum of squared feature values equals 1 (unit vector). Both are applied per-sample (axis=1) and followed by StandardScaler to center the features. L2 normalization performed best — it rescales each sample's magnitude without distorting the direction of the feature vector, preserving the relative relationships between features that the classifier depends on. Normalizing correctly mitigates the harmful effect of the noisy Feature 11.

## Key Findings — Bankruptcy Prediction

**Dataset:** UCI Taiwanese Bankruptcy Prediction (id=572) — 6,819 companies, 95 financial features
**Method:** 10-Fold Stratified Cross-Validation with Logistic Regression
**Goal:** Classify companies as bankrupt (1) or not (0) from financial ratios

| Scenario | Mean AUC |
|---|---|
| 1 Feature | low |
| 5 Features | moderate |
| 10 Features | higher |
| 11 Features (no normalization) | drops |
| 11 Features + L1 | recovers |
| **11 Features + L2** | **best** |

- **Why folds differ with 1 feature:** with only one financial ratio, the model is highly sensitive to which specific companies land in each fold — small random variation in the subset composition strongly affects performance.
- **Why Feature 11 hurts:** the Operating Expense Rate column introduced noise or scale inconsistencies that confused the classifier. Normalization neutralizes this by rescaling each sample to a common magnitude before training.
- **Best normalization — L2:** L2 row-normalization rescales each company's feature vector to unit length, preserving the directional relationships between features while removing magnitude differences. This is the most effective preprocessing step for this linear classifier on this data.

> **Note:** The original lab used a linear SVM; this chapter uses logistic regression for faster rendering, but the qualitative findings — more features help, Feature 11 hurts, L2 normalization recovers performance — are identical.
